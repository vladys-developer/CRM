# Principios de Diseño de APIs

Domina los principios de diseño de APIs REST y GraphQL para construir interfaces intuitivas, escalables y mantenibles que encanten a los desarrolladores y resistan el paso del tiempo.

## Usa esta habilidad cuando

- Diseñes nuevas APIs REST o GraphQL.
- Refactorices APIs existentes para mejorar su usabilidad.
- Establezcas estándares de diseño de APIs para tu equipo.
- Revises especificaciones de APIs antes de su implementación.
- Migres entre paradigmas de API (de REST a GraphQL, etc.).
- Crees documentación de API amigable para desarrolladores.
- Optimices APIs para casos de uso específicos (móvil, integraciones de terceros).

## No uses esta habilidad cuando

- Solo necesites una guía de implementación para un framework específico.
- Estés realizando trabajo solo de infraestructura sin contratos de API.
- No puedas cambiar o versionar las interfaces públicas.

## Instrucciones

1. Define los consumidores, casos de uso y restricciones.
2. Elige el estilo de API y modela los recursos o tipos.
3. Especifica errores, versionado, paginación y estrategia de autenticación.
4. Valida con ejemplos y revisa la consistencia.

Consulta `resources/implementation-playbook.md` para obtener patrones detallados, listas de verificación y plantillas.
